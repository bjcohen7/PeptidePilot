# PeptidePilot

Codex workspace for building and migrating the PeptidePilot funnel.

## Local Development

```bash
corepack pnpm install
cp .env.example .env
corepack pnpm dev
```

Local app URL:

```text
http://localhost:3000/
```

## Verification

```bash
corepack pnpm check
corepack pnpm test
corepack pnpm build
```

## Environment

Copy `.env.example` to `.env` for local work. Do not commit `.env`.

Required for full admin persistence:

- `DATABASE_URL`
- `ADMIN_EMAILS`
- `JWT_SECRET`
- `OAUTH_SERVER_URL` and `VITE_APP_ID` when using the Manus OAuth flow

Useful production values:

- `SITE_URL`
- `META_CAPI_TOKEN`
- `WEBHOOK_TIER1_URL`
- `WEBHOOK_TIER2_URL`
- `WEBHOOK_TIER3_URL`

## Database

Generate migrations only when the Drizzle schema changes:

```bash
corepack pnpm db:generate
```

Apply existing migrations to the configured database:

```bash
corepack pnpm db:migrate
```

After migration, sign into `/admin/partners` and use **Seed Legacy Links** to move hard-coded vendor links into managed affiliate rows.

Current status:

## Migration Notes

- The original Manus zip is preserved at `imports/peptidematch.zip`.
- An untouched extracted copy is preserved at `imports/peptidematch-extracted/`.
- The working app has been copied into the project root.
- The public funnel is wired as landing page, quiz, processing screen, results page, blog, and legal pages.
- The live sitemap is preserved at `imports/live-sitemap.xml`.
- The missing PSEO layer has been rebuilt from the live sitemap as data-driven routes under `client/src/data/pseo.ts`.
- Restored PSEO sections include `/peptides`, `/goals`, `/compare`, `/stacks`, `/guides`, `/for`, and `/reviews`.
- The route inventory now lives in shared code at `shared/pseo.ts` so the UI and server sitemap use the same source.
- `/sitemap.xml` and `/robots.txt` are generated by the Express server.
- Lead capture stores quiz submissions through the `quiz.submitQuiz` tRPC mutation when `DATABASE_URL` is configured.
- Affiliate clicks are tracked through the `quiz.trackAffiliateClick` tRPC mutation when `DATABASE_URL` is configured.
- Affiliate partner management has a first-pass admin route at `/admin/partners`.
- Affiliate partner/link tables have been added for future persistence: `affiliate_partners` and `affiliate_links`.
- The affiliate admin includes an assistant command box with preview/confirm behavior for link changes.
- Affiliate links support `isGlobal` and `sortOrder` so a partner link can be applied globally and pinned to position #1.
- Legacy vendor links from the scoring engine can be seeded into managed affiliate links from the admin panel.
- Affiliate changes are written to `affiliate_audit_events`.
- Owner notification is non-fatal when the old Manus/Forge notification env vars are missing.

## Known Gaps

- OAuth/admin environment variables are not configured locally yet.
- Admin routes are wired, but local sign-in still depends on OAuth environment configuration.
- The Manus export did not contain PSEO source files; current PSEO pages are restored route/content scaffolds generated from the live sitemap URL inventory.
- Analytics placeholders from the Manus export were removed from `client/index.html`; analytics should be reintroduced intentionally.
